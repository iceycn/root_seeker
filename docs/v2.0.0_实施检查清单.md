## 启用 AI 驱动主流程

在 `config.yaml` 中设置：

```yaml
ai_driven_enabled: true
```

启用后，分析请求将优先走 AiOrchestrator（Plan → Act → Synthesize → Check + 下一轮决策），支持多轮迭代（每轮结束由 AI 决策是否继续，最多 max_analysis_rounds 轮，默认 20）。任意 tool 失败或 Plan 解析失败时自动回退到直连路径。
