# v2.0.0 详细 Review：实现、设计、融合与衍生问题

## 一、本次修复：evidence.context_search 主循环不可用

### 问题

- **现象**：Plan 推荐路径 A 包含 `evidence.context_search`，但主循环调用该工具时返回 `{"found": false, "reason": "上下文尚未初始化或不在证据收集阶段"}`。
- **根因**：主循环的 `context` 仅包含 `trace_id`、`analysis_id`，未注入 `evidence_ctx`；而 `EvidenceContextSearchTool` 依赖 `context["evidence_ctx"]` 才能工作。
- **影响**：`evidence.context_search` 仅在 `_collect_evidence_recursive`（NEED_MORE_EVIDENCE 子计划）中可用，主循环中完全不可用。

### 修复

1. **主循环初始化 evidence_ctx**：每轮开始时创建 `EvidenceContext`，用 `prev_tool_results` 初始化，跨轮累积。
2. **注入 context**：`context["evidence_ctx"] = evidence_ctx`，供 `call_tool` 传入工具。
3. **增量更新**：每次成功工具调用后，若非 `evidence.context_search`，则 `evidence_ctx.add(tool_name, text)`；若为 `evidence.context_search` 且命中，则用 `to_evidence_text` 替换 tool_results 中的原始 JSON，与 `_collect_evidence_recursive` 行为一致。

### 代码变更

- `orchestrator.py` 主循环：创建 `evidence_ctx`、注入 `context`、工具成功后更新 `evidence_ctx` 并处理 `evidence.context_search` 结果替换。

## 三、设计层面 Review

### 3.1 evidence_ctx 生命周期

- **主循环**：每轮新建，用 `prev_tool_results` 初始化，本轮工具调用后增量更新。
- **递归证据收集**：每次调用新建，用传入的 `collected` 初始化，子计划内独立更新。
- **跨轮**：主循环的 `prev_tool_results` 包含上一轮全部结果，故 evidence_ctx 自然跨轮累积。

### 3.2 双路径一致性

- **主循环** 与 **_collect_evidence_recursive** 对 `evidence.context_search` 的处理一致：
  - 命中时用 `to_evidence_text` 替换原始 JSON，便于 Synthesize 直接使用。
  - 非 evidence 类工具结果均加入 evidence_ctx。

### 3.3 与 v2.0.0 计划差异（已修复）

| 差异项 | 修复 |
|--------|------|
| Plan 倾向早期 analysis.run | 提示词强化「勘探优先」；`_reorder_steps_cline_mode` 将前两步的 analysis.run 移至末尾 |
| Check 无一致性、可复现性 | `_check_and_sanitize` 增加可复现性检查（query_key 等）、一致性检查（勘探证据 vs 结论泛化） |
| Act 无结果压缩与可复现参数 | `_truncate_text` 支持 `repro_hint`；`_build_repro_hint` 为 correlation/index 类工具截断时保留 query_key/trace_id |

## 五、衍生问题与建议

### 5.1 低优先级

| 问题 | 建议 |
|------|------|
| contextModification 未使用 | 若需 Hook 注入上下文，可在 Plan 的 `build_plan_user_prompt` 或 Synthesize 的 `build_synthesize_user_prompt` 中追加 `pre_out.context_modification`。 |
| _try_check_extra_tools 固定工具 | 可改为由 Check 阶段 LLM 决定追加工具及原因（参考 v2.0.0 计划）。 |
| evidence_ctx 跨轮字符上限 | 当前 `max_evidence_total_chars` 单轮生效，跨轮时 `from_tool_results` 可能超出；`EvidenceContext.add` 有内部截断，影响有限。 |

- **主循环 evidence.context_search 集成测试**：Mock MCP，验证主循环中 evidence_ctx 注入与结果替换。
- **Plan 提示词**：继续弱化 analysis.run，强化 code.search/code.read/evidence.context_search 勘探路径。
